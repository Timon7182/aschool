# A School — сайт образовательного центра

Статический двуязычный (RU / KZ) сайт образовательного центра **A School** (Алматы).
Хостинг: **GitHub Pages**, базовый URL: `https://timon7182.github.io/aschool/`

## Публикация на GitHub Pages

1. Откройте репозиторий на GitHub.
2. Перейдите: **Settings → Pages**.
3. В разделе **Source** выберите:
   - Branch: `main`
   - Folder: `/ (root)`
4. Нажмите **Save**. Через 1–3 минуты сайт появится по адресу `https://timon7182.github.io/aschool/`.

### Свой домен (опционально)

Если хотите использовать собственный домен (например, `aschool.kz`):
1. Добавьте файл `CNAME` в корень репозитория с единственной строкой: `aschool.kz`
2. У регистратора домена создайте DNS-запись `CNAME` с указанием на `timon7182.github.io`.
3. В **Settings → Pages → Custom domain** укажите свой домен.
4. После этого обновите все `canonical`-ссылки и hreflang в HTML-файлах и `sitemap.xml` — замените `https://timon7182.github.io/aschool/` на `https://aschool.kz/`.

## Яндекс.Метрика — пошаговая настройка

### 1. Создание счётчика

1. Зайдите на [metrika.yandex.ru](https://metrika.yandex.ru) (войдите через Яндекс-аккаунт).
2. Нажмите **«Добавить счётчик»**.
3. Заполните:
   - **Имя:** A School
   - **Адрес сайта:** `timon7182.github.io/aschool` (или ваш домен)
   - **Часовой пояс:** Алматы (UTC+5)
4. Включите галочки:
   - **Вебвизор** — запись поведения пользователей на сайте
   - **Карта кликов** — тепловая карта кликов
   - **Карта скроллинга**
5. Нажмите **«Создать счётчик»**.

### 2. Вставка номера счётчика в код сайта

После создания счётчика Яндекс покажет его **номер** — например, `98765432`.

Откройте файл `assets/js/main.js`, найдите строку:

```js
const YM_COUNTER_ID = null; // TODO: вставить номер счётчика
```

Замените `null` на номер вашего счётчика:

```js
const YM_COUNTER_ID = 98765432;
```

После этого Метрика начнёт работать автоматически — код уже встроен в файл.

### 3. Цели для отслеживания конверсий

В интерфейсе Метрики перейдите: **Настройка → Цели → Добавить цель**.

Создайте 5 целей с типом **«JavaScript-событие»** и следующими идентификаторами (скопируйте точно, как написано):

| Название цели                  | Идентификатор (Условие)  |
|-------------------------------|--------------------------|
| Отправка формы                | `form_submit`            |
| Клик по WhatsApp              | `whatsapp_click`         |
| Клик по телефону              | `phone_click`            |
| Клик по соцсети               | `social_click`           |
| Открытие формы записи         | `trial_modal_open`       |

Эти идентификаторы уже используются в коде `main.js` — при соответствующих действиях пользователя цели будут засчитываться автоматически.

### 4. Проверка работы Метрики

Добавьте к URL сайта параметр `?_ym_debug=1`, например:
```
https://timon7182.github.io/aschool/?_ym_debug=1
```
Откройте консоль браузера (F12 → Console). Если Метрика работает, вы увидите сообщения с `[YM]`. Совершите действие (кликните по кнопке «Записаться»), и в консоли появится `[YM] reachGoal trial_modal_open`.

### 5. Связь с Яндекс.Директом

Для корректной аналитики рекламных кампаний:
1. Яндекс.Метрика и Яндекс.Директ должны быть привязаны к **одному Яндекс-аккаунту**, либо вы должны предоставить доступ к Метрике из Директа.
2. В Директе при создании объявлений используйте **разметку ссылок** (опция «Размечать ссылки» в настройках кампании) — это автоматически добавит параметр `yclid` к URL.
3. В `robots.txt` уже прописан `Clean-param` для параметров `utm_*` и `yclid` — поисковик не будет дублировать страницы с рекламными метками.
4. В настройках кампании Директа укажите **ключевые цели** из Метрики — выберите `form_submit` и `whatsapp_click` как конверсионные цели для оптимизации ставок.

## Настройка формы заявок

### 1. Телефон WhatsApp

В файле `assets/js/main.js` найдите:
```js
const WHATSAPP_PHONE = '77001234567'; // TODO
```
Замените на реальный номер в формате: `7XXXXXXXXXX` (без `+`, пробелов и скобок).

### 2. Email для дублирования заявок (через formsubmit.co)

В файле `assets/js/main.js` найдите:
```js
const LEAD_EMAIL = 'info@aschool.kz'; // TODO
```
Замените на реальный email, куда будут приходить копии заявок.

### 3. Активация formsubmit.co

При **первой** отправке формы на указанный email придёт письмо от formsubmit.co с просьбой подтвердить адрес. Перейдите по ссылке в этом письме — после этого дублирование заявок на email заработает. Если письмо не пришло — проверьте папку «Спам».

Весь код помечен комментарием `// TODO` или `<!-- TODO -->`.

| Что заменить | Где найти | Что сделать |

| WhatsApp-номер `77001234567` | `assets/js/main.js` → `WHATSAPP_PHONE` | Заменить на реальный номер |

| Фото учеников | Блоки `<div class="photo-slot">` во всех HTML | Заменить CSS-заглушки на реальные фотографии центра |
| Фото педагогов | `index.html`, секция «Педагоги» | Вставить реальные портреты педагогов |
| Логотип | `assets/img/logo.svg` | Заменить (инструкция ниже) |
| Отзывы | `index.html`, секция «Отзывы» | Вставить реальные отзывы родителей |
| Ссылка на WhatsApp в хедере | `index.html`, `kz/index.html` | Обновится автоматически через `WHATSAPP_PHONE` в main.js |

### Как заменить логотип

1. Получите от клиента оригинальный логотип в формате PNG или SVG.
2. Сохраните файл как `assets/img/logo.svg` (или `logo.png`).
3. Если используете PNG, измените в HTML-файлах атрибут `src` у тегов `<img>` с логотипом:
   ```html
   <!-- было -->
   <img src="assets/img/logo.svg" ...>
   <!-- стало -->
   <img src="assets/img/logo.png" ...>
   ```
4. Для favicon: сохраните значок как `assets/img/favicon.svg` (или `.png`) и при необходимости обновите `<link rel="icon">` в `<head>` каждой страницы.

## SEO-чеклист после публикации

### Яндекс.Вебмастер

1. Зайдите на [webmaster.yandex.ru](https://webmaster.yandex.ru).
2. Нажмите **«Добавить сайт»**, введите `https://timon7182.github.io/aschool/`.
3. Подтвердите владение сайтом — выберите метод **«Мета-тег»**: скопируйте предложенный тег `<meta name="yandex-verification" ...>` и вставьте его в `<head>` файла `index.html`.
4. После подтверждения: **Индексирование → Sitemap-файлы** → добавьте `https://timon7182.github.io/aschool/sitemap.xml`.
5. Проверяйте раздел **«Качество сайта»** и **«Диагностика»** раз в неделю.

### Google Search Console

1. Зайдите на [search.google.com/search-console](https://search.google.com/search-console).
2. Добавьте ресурс — тип **«URL-prefix»**, введите `https://timon7182.github.io/aschool/`.
3. Подтвердите владение (рекомендуется метод HTML-тег: вставить в `<head>` тег `<meta name="google-site-verification" ...>`).
4. Перейдите в **Sitemaps** → добавьте `https://timon7182.github.io/aschool/sitemap.xml`.

### Локальная выдача (карты и справочники)

Для детского центра критически важна локальная выдача — родители ищут «подготовка к школе рядом».

1. **Яндекс.Бизнес** ([business.yandex.ru](https://business.yandex.ru)) — добавьте организацию: название, адрес, телефон, сайт, фото центра, категория «Образовательный центр». После модерации (1–5 дней) центр появится на Яндекс.Картах.
2. **2ГИС** ([2gis.ru/partners](https://2gis.ru/partners)) — бесплатное добавление организации. Популярен в Казахстане, важен для Алматы.
3. **Google Business Profile** ([business.google.com](https://business.google.com)) — заполните профиль: название, адрес, часы работы, фото, ссылка на сайт. После верификации центр появится на Google Картах и в блоке «Local Pack» в поиске.

**Важно:** в профилях всех трёх сервисов используйте одинаковые название, адрес и телефон (NAP-консистентность) — это улучшает ранжирование в локальной выдаче.
